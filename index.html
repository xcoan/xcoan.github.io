Professional History

I grew up in Michigan City, Indiana, where I attended the local high school. During my Junior year, I worked an internship for the school system IT department, where I set up many of the computer labs for the schools and pushed disk images to computers. I also aided in maintaining server-switch boxes the school's network ran on, which included replacing batteries and electrical connectors, and replacing network cables.


In the summer of 2016, I took part in the Xtern Program in Indianapolis, Indiana. Through the Xtern Program, I met 130 other college students who were interested in working in computing fields. Thanks to Xtern, I had an amazing internship with Octiv.

At Octiv, my work was focused around Ruby on Rails and Ember.js. Over I worked extensively with I18N and L10N with YAML files to allow the application to be translated and localized into various languages around the world. Using a chron job, I modified our internal API to automatically purge month-old data from our Redis NoSQL data cache. I also worked on preventing potential XSS and CSRF attacks.

I wrote a blog post about my experience with Xtern and Octiv on the octiv website, which you can view here.


Featured Projects

Local Record data aggregation and visualization

A web app written for the 2016 Indianapolis Civic Hackathon, and is still under active development. It is built using Node Express on top of MongoDB. Using the Google Maps API and a GEOJson file recieved via an AJAX call to OpenIndy, the app plots data of crimes onto a map of Indianapolis, allowing for efficient and effective data aggregation and visualization for citizens and lawmakers alike. Local Record recieved second place in the hackathon. Work on local record was done in collaboration with fellow hacker Aaron Krueger.

Medical API record keeping for medical professionals

Developed in November 2016 for ArchHacks. The Medical API is a server system built on Rails that allows Hospitals and other medical institutions to easily track basic information on their patients. The goal of the application is to minimize the amount of time medical professionals spend time doing paperwork, allowing more time to see patients and take care of emergencies.
